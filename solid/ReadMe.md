# Solid

Nada mais sao do que principios de desenvolvimento de software

O solid e essencial pra combater 4 problemas no software

- rigidez

Sistemas onde mudancas simples se tornam complexas e caras

- imobilidade

Componentes dificeis de reutilizar em outros contextos ou projetos

- Fragilidade 

Codigo que quebra inesperadamente quando modificado em uma parte especifica

- Acoplamento excessivo

Modulos fortemente dependentes, dificultando manutencao e evolucao

## Consideracoes finais 

- quanto mais perto do core da aplicacao mais voce cria abstracoes

- quanto mais pra ponta, mais voce precisa aplicar funcoes concretas, voce nao precisa de milhoes de abstracoes
